Sale of tokens

The entire size of the token offering is 88,888,888 tokens at the total value of $25,000,000. The purchasing price of the token will be $0.443786961 (~$0.44). There will be four TOKEN SALE rounds and 4 JACKPOT rounds. In the figure below one can see the issue amounts each time and for each type of issuance. During the second of TOKEN SALE there is 1025% increase in the amount issued, while between the second and the third this number is roughly 50%, while between the third and fourth it is 103%. On the other hand, the increase in the number of tokens distributed through JACKPOT between JACKPOT 1 and 2, 2 and 3, and, 3 and 4 is 338%, 129%, and 113%, respectively.

In addition, the issuers of the token will possess a separate nominal amount of 29,629,629 tokens that will not be available to the public throughout the issuance period. Each TOKEN SALE takes place three weeks between each other while the JACKPOTS are on the Monday of the week following each TOKEN SALE.

After every token sale, there is a jackpot game taking place: every token that has been issued during the TOKEN SALE (hereafter referred to as a “sold token”) has the chance of winning every (NOT ALL) token that is being JACKPOTED. The JACKPOT procedure works as follows:

  • All the sold tokens (basically, their IDs) are assigned with a number that represents the jackpot factor.
  • The jackpot factor represents the number of occurrences of the sold token’s ID in the jackpot list. In other words, it represents the chance of a given sold token to be picked as the winner of the jackpot token.
  • The jackpot list is a simple list over which we run a model that randomly picks one ID from it.
  • This ID determines the winner of the jackpotted token: the one who holds the token with this ID (the sold token) is receiving the jackpotted token.
  • We iterate this procedure for every of the jackpotted tokens.
  • The table above serves to illustrate this: there are ten groups of sold tokens for every round of funding. The distribution of the jackpot factors for every round is the same and follows the exponential distribution with the decay constant of 0.8888 (see the spreadsheet for details). This is represented by the column “Distr. Tokens”. The jackpot factors themselves, however, are different for every token sale. These are represented by the respective columns in the table above. As it is evident, those who participate in the early rounds of token sales will enjoy holding sold tokens with higher average jackpot factors. The advantages of these are twofold:
    1. An average sold token from earlier token sale rounds is having larger chance of winning a jackpot token
    2. The sold tokens from earlier token sale rounds are participating in more jackpot rounds than those issued later (i.e. the token issued during the first financing round is participating in 4 jackpot rounds, the second in three etc.)
  • So, overall, intuitively, the procedure is as follows:
    1. The list of sold tokens is being fulfilled with their IDs. The number of occurrences of every token ID in this list is determined by the jackpot factor (I hope fractional jackpot factors are not a problem).
    2. For every jackpot token there is a procedure coded into the smart contract that randomly picks one ID from the list. The holder of this token receives a jackpot token.
    3. This procedure iterates over every jackpot token.
    4. When the new tokens from the next token sale round are all sold, their IDs are added to the list (with the respective jackpot factors).
    5. For every jackpot token of the new jackpot round the procedure again iterates over all the sold tokens.
    6. Repeat.

All the details can be found in the spreadsheet.
